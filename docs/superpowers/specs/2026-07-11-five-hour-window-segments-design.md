# 5時間ウィンドウの経過表示設計

日付: 2026-07-11

対象: Token Checker のポップオーバー

## 目的

5時間のレート制限欄にも、週次欄の青い7分割バーと同じ経過時間表示を追加する。

5時間欄では青いバーを5分割し、1本を1時間として表示する。

使用率を示す既存の緑、橙、赤のバーは変更しない。

## 表示仕様

5時間欄の使用率バーの下に、青い5分割バーを配置する。

各セグメントは次の規則で塗る。

- 経過済みの1時間は全体を青く塗る。
- 現在進行中の1時間は、経過割合に応じて左から部分的に青く塗る。
- 未経過の時間はグレーのトラックだけを表示する。
- 5時間ウィンドウの開始前は、5本すべてを未経過として表示する。
- リセット時刻以後は、5本すべてを青く塗る。

たとえば、ウィンドウ開始から2時間30分が経過した場合の塗り率は `[1, 1, 0.5, 0, 0]` になる。

5分割バーの高さ、間隔、青色、背景色、角丸は週次の7分割バーと揃える。

週次欄だけに表示しているリセット日時は現状を維持し、5時間欄の見出しには追加しない。

## 時間計算

既存の週次専用計算を、任意のセグメント数と1セグメントの時間を受け取る純粋な計算処理へ共通化する。

5時間表示ではセグメント数を5、1セグメントを1時間として渡す。

週次表示ではセグメント数を7、1セグメントを1日として渡す。

ウィンドウ開始時刻と塗り率は次の式で求める。

```text
windowLength = segmentCount * segmentDuration
windowStart = resetsAt - windowLength
elapsedSegments = clamp((now - windowStart) / segmentDuration, 0, segmentCount)
segmentFill[i] = clamp(elapsedSegments - i, 0, 1)
```

計算結果はセグメント数と同じ要素数の配列にする。

この共通化により、5時間表示と週次表示の境界処理を同じ規則に保つ。

## ビュー構成

週次専用名になっている分割バービューを、5時間と週次の両方で使える名前と入力へ変更する。

分割バービューは塗り率の配列を受け取り、配列の要素数と同じ本数を描画する。

5時間欄には5要素、週次欄には7要素の塗り率を渡す。

5時間欄と週次欄で異なるのはセグメント数と時間単位だけであり、描画処理は共有する。

既存の使用率、残り時間、週次のリセット日時、Sonnet週次表示、メニューバー表示は変更しない。

## テスト

時間計算の単体テストでは、少なくとも次の境界を確認する。

- 5時間ウィンドウ開始時は `[0, 0, 0, 0, 0]` になる。
- 2時間30分経過時は `[1, 1, 0.5, 0, 0]` になる。
- リセット時刻とリセット後は `[1, 1, 1, 1, 1]` になる。
- リセット時刻が5時間より先の場合は `[0, 0, 0, 0, 0]` になる。
- 既存の7日ウィンドウの計算結果が変わらない。

実装はテストを先に追加し、対象テストが期待どおり失敗することを確認してから行う。

実装後は `swift test` と `swift build` を実行する。

## 受け入れ条件

1. ClaudeとCodexの5時間欄に、青い5分割バーが表示される。
2. 5本の各セグメントが1時間を表し、現在の1時間は経過割合に応じて部分塗りされる。
3. 週次の青い7分割バーは従来どおり表示される。
4. 既存の使用率バー、表示テキスト、Sonnet週次行、メニューバー表示に変更がない。
5. `swift test` と `swift build` が成功する。
